edna -- an MP3 server

 Quickstart: download

This is a quick hack that I put together, based on ideas from my friend, Geoff Elliott. He had written a program named ampRadio that operates much like edna -- serving up HTML pages with links to MP3 files, and then serving out those MP3s.

It isn't ready for prime time, but may have information that interests you. We would also welcome any feedback about the site. Please feel free to sign up on the edna mailing list.

edna is currently labeled as version 0.3 because there are still a few more features that I would like to add. It is very stable and can be used (now) without a problem.

While any silly web server can do this (serve up MP3s), there are two cool features about edna:

  • The pages are dynamically constructed, adjusting to directory structure and the files in those directories. This is much nicer than using simple directory indexing. While the dynamic file list could be done with various CGI or PHP-like tools, the dynamic directories would be a lot harder.
  • This is the coolest part... Rather than directly serving up an MP3, the software serves up a playlist. This gets passed to your player (e.g. WinAmp) which turns around with an HTTP request to stream the MP3. I must give credit to Geoff for this one :-)

When constructing a page for a directory, edna will list all subdirectories, MP3 song files (.mp3 extensions), playlists (.m3u extensions), and certain other types of music files (such as .wav). All other files will be ignored.

edna can automatically create play lists for individual directories of songs, and all directories recursively. These lists can also be shuffled (by edna, rather than your player).

Your MP3 player must be able to recognize .m3u files, and your browser must know to pass .m3u files to your player. Your player must be capable of handling URLs within a playlist. I've tested this with WinAmp, which definitely meets these needs.

I'd also like to thank Andy Lowe for submitting the patches for recursive play lists. Bill Tutt has provided some excellent patches, too; version 0.4 will include his patches to turn edna into a Win32 Service.

Can I have the source?

Sure. edna is licensed under the GPL. It is a small Python script. At the moment, it is about 500 lines (version 0.1 was only 250!). You can view the thing online, or grab the distribution. edna is checked into a CVS repository and is available for viewing with your web browser.

The server should work for any platform. I've tested it on Linux and Windows. I've been using Internet Explorer and WinAmp (on Windows) to navigate the pages and play the files.

Installation and Use

Using edna is boringly simple. Make sure you have Python 1.5 installed on your system. Edit the edna.conf file with your setup. This is simple to do, as all you need to tell it is:

  • what port to run on
  • the directories for your MP3 files
  • optionally, how to log requests

Then, simply run edna.py. This can be done using:

% python edna.py

It will then go into an infinite loop, serving up pages and files. Hit ctrl-C to stop it.

Modifications

If you don't like the HTML it generates, then look at edna.py. You should be able to figure out how to change it.

I'm quite willing to accept patches or other fixes. Just email them to me at gstein@lyra.org.

edna has some code in it to extract ID3 tags from MP3 files. I'm not using it yet, but will add some code to display the data on the generated web pages. I might also look into using some kind of template mechanism for generating the web pages. A number of people have requested access lists to protect their files, so I'll be adding that soon, too. If you have a pet feature, then please let me know!

Threading vs Forking

By default, edna will use a threading mechanism to serve out requests. If threading support is not available on your platform (the OS doesn't support it, or Python was built without it), then edna will default to forking processes to serve requests. No big deal, but I mention it since edna-0.1 did not fall-back to forking when threading was not available. As of edna-0.2, it will figure out the right thing to do.
